## 编码规约说明

### 普通类

与平常一般的类无区别。
继承自Object，可以单独使用。
若继承抽象类(以A开头)，则不直接继承Object。
类统一为大驼峰命名法，但最前方可能会加特殊标识。

### I开头的类

I开头意思是Interface，为接口类。
不继承Object，内部没有数据，只有虚函数。
其他任意类可以继承多个并提供接口实现。

### A开头的类

A开头意思是Abstract，为抽象类。
继承自Object，内部有数据、函数、虚函数。
只有普通类可以单独继承一个。

### C开头的类

C开头在我这里的意思是Component。
内部有数据、函数、虚函数(但没有必要)。
是专门用于多继承的组件类，不会单独直接使用。
也不会继承自Object。
构造器、析构器为受保护访问。

### T开头的类

T就是template，模板类的意思
我通常需要再次typedef限定一下某些模板类的类型参数
以免一些没注意到的问题弄出bug

### 函数

函数统一为大驼峰命名法，
每个词汇不分割，但以大写开头。

### 变量

变量统一为小驼峰命名法，
与函数的区别就是第一个字母小写。

3. 下载文件夹路径下执行make.exe
